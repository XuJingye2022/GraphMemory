# CONTEXT

本项目的领域术语表。不包含实现细节。

## 核心实体

- **节点 (Node)**
  - **事实性节点 (Factual Node)**: 可独立成立的陈述或知识点。相似者可合并。
  - **流程性节点 (Procedural Node)**: 流程/工作流中的单个操作步骤。禁止合并，每个步骤是独立可替换的单元。分支替代通过在出错步骤旁添加纠正节点实现。

- **边 (Edge)**: 从源节点到目标节点的有向联想。每条边同时具备正向强度和反向强度（独立）。

- **联想类型 (RelationType)** — 枚举约束，不可自由填写:
  - `sequence` — 流程步骤的先后顺序。检索权重最高。
  - `alternative` — 流程步骤的替代方案。按需触发。
  - `causes` — 因果关联。
  - `contains` — 包含/分解关系。
  - `similar_to` — 事实性节点间相似（供合并使用）。
  - `correction` — 错误节点指向正确节点。按需触发。
  - `generic` — 兜底，语义不明确。权重最低。

- **抑制边 (Inhibition)** — 边上的 `inhibition` 字段，非零时覆盖正向联想，阻断从 source 到 target 的激活。建立条件：用户明确禁止、安全违规、逻辑矛盾。

- **标签 (Tag)** — 节点上的自由标签（字符串数组），无预定义枚举。用于粗筛：先按 tag 过滤候选集，再用 embedding 语义排序。Agent 自行维护自己常用的标签词汇。

## 运行时概念

- **工作记忆 (Working Memory)** — 当前推理上下文中活跃的节点集合。对应 LLM 的上下文窗口。容量极小，由加载分数（load_score）选出。

- **加载分数 (Load Score)** — 决定节点是否进入工作记忆的计算值。由核心度 + 任务相关性 + 边平均强度加权求和。

- **思维链 (Chain)** — 一次推理或任务执行中，按顺序访问的节点和边的完整路径。存储为 `{"nodes": [...], "edges": [...], "query": "触发问题"}`。保留 3 个月后按时间全删。

- **重放 (Replay)** — 按 Agent 身份定义（agent_profile.yaml）挑选核心路径，模拟遍历但不实际执行，只强化路径上的节点核心度和边强度。目的：防止与 Agent 职责相关的记忆因长期不使用而衰减。
